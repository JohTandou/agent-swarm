## Role

The UX/UI audit analyzes the entirety of the application's visual and interactive experience. It evaluates design system foundations, typography, color palette, layout composition, component quality, animations and micro-interactions, accessibility (WCAG), responsive design, state management (loading, empty, error), copywriting and navigation architecture. It also measures perceived performance.

## Use Cases

- **Design overhaul**: establish a baseline before a redesign
- **Accessibility audit**: verify WCAG compliance of an interface
- **Conversion optimization**: identify friction points in the user journey
- **Quality review**: benchmark UI/UX quality against Apple-grade standards

## Triggers

The planner agent loads this skill when:
- The user wants a complete interface and user experience audit
- A question about visual quality, accessibility or design system is asked

## Inputs

- Application interface (pages, components, navigation)
- Existing design system (tokens, variables, guidelines)
- Design mockups or specifications if available

## Outputs

- Structured audit report by dimension (design system, typography, palette, etc.)
- List of accessibility violations with WCAG fixes
- Prioritized recommendations to improve UI/UX quality

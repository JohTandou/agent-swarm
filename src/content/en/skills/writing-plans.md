## Role

The writing-plans skill transforms specifications or requirements into detailed implementation plans. Before a single line of code is written, it analyzes needs, breaks down the work into atomic steps, identifies dependencies, estimates complexity, and produces a structured plan document that will serve as a roadmap for execution agents.

## Use Cases

- **New feature**: plan the implementation before coding
- **Complex specification**: break down a requirements document into manageable tasks
- **Estimation**: evaluate the effort needed before committing
- **Team alignment**: share a clear plan with all stakeholders

## Triggers

The planner agent loads this skill when:
- A multi-step specification or set of requirements is provided
- The user wants to plan before implementing

## Inputs

- Functional or technical specifications
- Project context (architecture, stack, conventions)
- Constraints (deadlines, resources, external dependencies)

## Outputs

- Structured implementation plan with atomic steps
- Identified dependencies between steps
- Complexity estimation and attention points
- Success criteria for each step

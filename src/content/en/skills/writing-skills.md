## Role

The writing-skills skill accompanies the creation, editing and verification of OpenCode skills. It guides the user in defining the skill's scope, writing instructions, structuring content, and validation before deployment. It ensures each skill is well-formed, consistent with conventions, and ready to be used by agents.

## Use Cases

- **New skill**: create a custom skill for a specific task
- **Maintenance**: edit an existing skill to improve or fix it
- **Validation**: verify a skill is correct before deploying it
- **Standardization**: ensure all skills follow the same conventions

## Triggers

The planner agent loads this skill when:
- The user creates, edits or verifies an OpenCode skill
- A new skill must be validated before deployment

## Inputs

- Skill specification (objective, context, use cases)
- Existing skill code or content (if editing)
- Skill writing conventions

## Outputs

- Skill created or edited, compliant with conventions
- Validation report (structure, content, consistency)
- Skill ready for deployment

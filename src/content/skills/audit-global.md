## Rôle

L'audit global effectue une revue exhaustive de l'application sur six dimensions critiques : la complétude d'implémentation (ce qui est livré vs ce qui est prévu), la readiness pour la production, la sécurité, la performance, la qualité du code et le marketing/SEO. Il produit un rapport consolidé avec des recommandations actionnables.

## Cas d'usage

- **Checklist pré-lancement** : vérifier qu'une application est prête pour la mise en production
- **Due diligence technique** : évaluer un projet existant avant reprise ou investissement
- **Amélioration continue** : identifier les axes d'amélioration prioritaires
- **Revue de code transversale** : coupler qualité code, performance et sécurité en une seule passe

## Déclencheurs

L'agent planner charge ce skill quand :
- L'utilisateur demande un état des lieux complet de son application
- Une demande d'audit global ou de "health check" est formulée

## Entrées

- Codebase complète (frontend, backend, configuration, CI/CD)
- Fichiers de spécifications et documentation
- Métriques de production si disponibles

## Sorties

- Rapport d'audit multi-dimensions avec scores par axe
- Liste priorisée des corrections (critiques, majeures, mineures)
- Recommandations stratégiques pour la mise en production

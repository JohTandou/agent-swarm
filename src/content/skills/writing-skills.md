## Rôle

Le skill writing-skills accompagne la création, l'édition et la vérification des skills OpenCode. Il guide l'utilisateur dans la définition du périmètre du skill, la rédaction des instructions, la structuration du contenu, et la validation avant déploiement. Il s'assure que chaque skill est bien formé, cohérent avec les conventions, et prêt à être utilisé par les agents.

## Cas d'usage

- **Nouveau skill** : créer un skill personnalisé pour une tâche spécifique
- **Maintenance** : éditer un skill existant pour l'améliorer ou le corriger
- **Validation** : vérifier qu'un skill est correct avant de le déployer
- **Standardisation** : s'assurer que tous les skills suivent les mêmes conventions

## Déclencheurs

L'agent planner charge ce skill quand :
- L'utilisateur crée, édite ou vérifie un skill OpenCode
- Un nouveau skill doit être validé avant déploiement

## Entrées

- Spécification du skill (objectif, contexte, cas d'usage)
- Code ou contenu existant du skill (si édition)
- Conventions de rédaction des skills

## Sorties

- Skill créé ou édité, conforme aux conventions
- Rapport de validation (structure, contenu, cohérence)
- Skill prêt pour le déploiement
